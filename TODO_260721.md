## 5. Coverage gaps — what the report cannot yet catch

Folded in from `TODO_260720_gaps.md`. The goal: no upstream failure reaches
bioinformatics wearing a PASSED label. This is the inventory of what the tool
currently cannot see, ordered by what it costs to close. Facts and costs only —
which are worth building is your call.

### 5a. Already parsed, never checked

The tool reads 24 metrics and gates on 8. These 16 are parsed, carried into
`wgsTriage_samples.tsv`, and never compared against anything. Gating any of them
costs one row in `THRESHOLDS` plus a threshold you are willing to defend; the
background archive can supply observed ranges for all of them, the same way it
did for the current 7.

| Metric | Source field | Failure class it would catch |
| --- | --- | --- |
| `pctAdapter` | `PCT_ADAPTER` | Adapter read-through; short inserts, library prep |
| `pctExcDupe` | `PCT_EXC_DUPE` | Library complexity; over-amplification of too little input |
| `pctMapped` | `reads_mapped_percent` | Wrong reference, contamination, failed run |
| `pctReadsAligned` | `PCT_PF_READS_ALIGNED` | Same, from the second tool |
| `pct30x` | `PCT_30X` | Coverage uniformity; mean depth hiding cold regions |
| `sdCoverage` | `SD_COVERAGE` | Coverage evenness; GC bias, capture artefacts |
| `interChromRate` | `pairs_on_different_chromosomes` | Third independent view of the chimera defect |
| `strandBalance` | `STRAND_BALANCE` | Strand-specific artefact |
| `pctImproperPairs` | `PCT_PF_READS_IMPROPER_PAIRS` | Overlaps `pctProperlyPaired`; confirm before adding |
| `pctExcMapq` | `PCT_EXC_MAPQ` | Bases lost to low mapping quality |
| `medianCoverage` | `MEDIAN_COVERAGE` | Mean/median divergence indicates skew |
| `totalReads`, `meanReadLength`, `meanAlignedLength`, `alignedFrac`, `insertSizeAverage` | various | Descriptive or already used elsewhere |

- [ ] Decide which of these become checks. Each needs a threshold, and a
      threshold with no basis is worse than no threshold. Do this against the
      rebuilt background (§1), which supplies the observed ranges.

### 5b. Delivered versus usable

`pctReadUsed` works because it compares what arrived against what survived.
Nothing else in the report does that. These would.

- [ ] **Usable depth against raw depth.** `MEAN_COVERAGE` is already post-filter;
      `PCT_EXC_TOTAL` says how much was excluded. Raw depth is recoverable from the
      two, so the report can state both: the depth delivered and the depth callable.
- [ ] **Usable yield against delivered yield.** `TOTAL_READS` x `MEAN_READ_LENGTH`
      is the yield that was billed; `TOTAL_READS` x `MEAN_ALIGNED_READ_LENGTH`, less
      the excluded fraction, is what reached a callable base. One number, in Gb, on
      the same row.
- [ ] **Genome actually covered.** `PCT_30X` against nominal depth: a 40x mean with
      a low `PCT_30X` was not a 40x genome.

### 5c. Needs data the Map stage does not produce

- [ ] **Sample swap / mislabelling.** Genotype concordance — Conpair, NGSCheckMate
      or somalier. The one class of failure this gate structurally cannot see.
      (This is the item formerly in §5; it needs more than mapping metrics.)
- [ ] **Cross-sample contamination.** VerifyBamID or ContEst; estimates a
      contamination fraction from SNP allele fractions.
- [ ] **Tumor-in-normal contamination.** Paired genotype analysis. Silently
      destroys somatic calling.
- [ ] **Reference genome mismatch.** BAM header `@SQ` lines against the expected
      reference. Cheap, but the header is not currently read.
- [ ] **Per-lane structure.** multiqc general stats is per read group; the tool
      deliberately reads only per-sample `.recal` rows. One bad lane averages away.
- [ ] **Insert size distribution shape.** Only the mean is read. Bimodality does
      not move a mean. The samtools `IS` block is unparsed — see §3, the mean-vs-
      median item, which reaches the same unparsed block.
- [ ] **Sex against metadata.** chrX/chrY coverage ratio; needs per-contig
      coverage.
- [ ] **Truncated BAM.** No EOF check, no read count against expectation.
- [ ] **GC bias.** `CollectGcBiasMetrics`, not run by the Map stage.
- [ ] **Index hopping.** Needs demultiplexing-stage data; not in any BAM.
- [ ] **Contaminating organism.** Screening; out of reach here.

  `pctReadUsed` limitation).

  `CollectInsertSizeMetrics` to the Map stage.
